# Github Rule
다음과 같은 Convetion을 따릅니다.

## Commit Convention
-   feat : 새로운 기능 추가
-   fix : 버그 수정
-   docs : 문서 추가 및 수정
-   style : 코드 포맷팅, 세미콜론 누락, 코드 변경이 없는 경우
-   refactor: 코드 리펙토링
-   chore : 빌드 업무 수정, 패키지 매니저 수정
